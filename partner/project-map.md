# Project Map

This map lists current calibration samples for the partner workspace. These projects are not the goal themselves; they are evidence surfaces for building a better agent-native working method.

## Current P0

Use real projects to improve the partnership across:

- problem definition;
- product and engineering judgment;
- context and memory design;
- contract-first implementation;
- verification and review;
- reflection and method improvement.

## Sample 1: Agent 化组织

Question:

- How should software express people, roles, projects, customers, requirements, actions, context, permission, and validation as a collaborative network?

Use as calibration for:

- agent-native product form;
- organizational object modeling;
- permissions and context boundaries;
- human-agent collaboration loops.

Current risk:

- Easy to become a grand architecture narrative without a narrow project experiment.

Partner behavior:

- Keep asking which object, workflow, or validation loop is being improved now.

## Sample 2: L0-L4 需求到交付链路

Question:

- How can AI compress the distance from formalized requirements to verifiable delivery without pretending to replace requirement clarification?

Use as calibration for:

- L0 problem definition;
- L1 structured expression;
- L2 contract;
- L3 engineering organization;
- L4 generated artifacts and evidence index.

Current risk:

- Downstream implementation may quietly patch upstream semantic gaps.

Partner behavior:

- Preserve human-owned L0-L2 and agent-owned L3-L4 boundaries unless the user explicitly changes the contract.

## Sample 3: Agent 知识治理闭环

Question:

- What makes agent knowledge reliable enough for real project work?

Use as calibration for:

- candidate knowledge capture;
- owner confirmation;
- active retrieval;
- source identity;
- confidence and degradation;
- answer traceability.

Current risk:

- Treating retrieval as the whole problem while ignoring write governance, confirmation, and failure modes.

Partner behavior:

- Require source-backed answers and explicit downgrade behavior when confidence or source quality is weak.

## Sample 4: Partner Workspace Itself

Question:

- What engineering carrier lets Codex act as a personalized human-agent peer over time?

Use as calibration for:

- local Markdown workspace;
- `AGENTS.md` contract;
- research and evidence tracking;
- decision log;
- later Skill extraction.

Current risk:

- Over-engineering into MCP or an application before the working method is stable.

Partner behavior:

- Keep v0.1 file-based, inspectable, and easy to revise.

## Sample 5: 端侧 Agent 数据权限回收

Question:

- How can an agent observe enough local context to help proactively, while recovering user data authority from centralized super apps instead of creating another opaque platform?

Use as calibration for:

- local-first context access;
- explicit permission boundaries;
- proactive nudge timing;
- source-backed personal workflow learning;
- human-held action authority.

Current risk:

- Proactive help can become silent surveillance or notification spam if observation sources, cooldown, and stop conditions are unclear.

Partner behavior:

- Treat proactive push as a candidate nudge with evidence first; require explicit approval before scheduling, cross-app access, or external notification.
